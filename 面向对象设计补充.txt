第三章补充：面向对象设计（可直接并入“详细设计”）

一、设计目标与分层
结合现有文档“AI智能助手 + 求职交流论坛 + 个人主页 + 法律案件与模拟演练”四大核心模块，建议采用“Controller（接口层）- Service（业务层）- Repository（数据访问层）- Domain（领域模型）”四层对象模型。

二、核心类（属性 + 方法）

1）用户与身份模块
（1）User
- userId: String（用户ID） // 用户唯一标识
- username: String（用户名） // 登录名/展示名
- passwordHash: String（密码哈希） // 加密后的密码
- role: String（USER/ADMIN） // 用户角色
- avatarImageUrl: String // 头像图片地址（支持jpg/png/webp）
- email: String // 邮箱
- phone: String // 手机号
- createdAt: DateTime // 创建时间
- updatedAt: DateTime // 更新时间
- 方法：
	- updateBaseInfo(username, avatarImageUrl, email, phone): void
	- changeRole(role): void
	- verifyPassword(rawPassword): Boolean
	- resetPassword(newPasswordHash): void

（2）UserProfile
- userId: String // 关联用户ID
- school: String // 学校
- major: String // 专业
- graduationYear: Integer // 毕业年份
- expectedCity: String // 期望城市
- expectedSalary: String // 期望薪资
- bio: String // 个人简介
- 方法：
	- updateProfile(school, major, graduationYear, expectedCity, expectedSalary, bio): void
	- validateProfileCompleteness(): Boolean
	- toResumeSummary(): String

（3）AuthSession
- token: String（JWT/会话令牌） // 会话凭证
- userId: String // 关联用户ID
- issuedAt: DateTime // 签发时间
- expiresAt: DateTime // 过期时间
- deviceInfo: String // 设备信息
- 方法：
	- isExpired(currentTime): Boolean
	- refresh(newExpiresAt): void
	- revoke(): void

2）AI智能助手模块
（4）ContractDocument
- documentId: String // 文档唯一标识

- originalText: String // 原始文本
- fileName: String // 文件名
- fileUrl: String // 文件存储地址
- uploadedAt: DateTime // 上传时间
- 方法：
	- updateSource(sourceType, originalText, fileName, fileUrl): void
	- getPlainText(): String
	- validateUpload(): Boolean

（5）RiskRule
- ruleId: String // 规则唯一标识
- ruleCode: String // 规则编码
- ruleName: String // 规则名称
- category: String（劳动合同/黑话识别/岗位风险） // 规则分类
- riskLevel: String（LOW/MEDIUM/HIGH） // 风险等级
- keywords: List<String> // 命中关键词
- legalBasis: String（法规依据） // 法律依据
- suggestionTemplate: String // 建议模板

- 方法：

（6）RiskItem
- riskItemId: String // 风险项唯一标识

- matchedText: String // 命中文本
- startIndex: Integer // 起始下标
- endIndex: Integer // 结束下标
- riskLevel: String // 风险等级
- reason: String // 风险原因
- legalBasis: String // 法律依据
- suggestion: String // 处理建议
- 方法：
	- updateSpan(startIndex, endIndex): void
	- updateReason(reason, legalBasis, suggestion): void
	- toHighlightSegment(): String

（7）AnalysisReport
- reportId: String // 报告唯一标识
- userId: String // 报告所属用户ID
- businessType: String（CONTRACT/BLACK_SLANG/JOB_RISK） // 业务类型
- inputSummary: String // 输入摘要
- score: Integer（0-100） // 风险评分
- overallRiskLevel: String // 总体风险等级
- riskItems: List<RiskItem> // 风险项列表
- createdAt: DateTime // 创建时间
- 方法：
	- addRiskItem(riskItem): void
	- calculateScore(): Integer
	- determineOverallRiskLevel(): String
	- toViewModel(): ReportViewModel

（8）AiQaMessage
- messageId: String // 消息唯一标识
- sessionId: String // 会话ID
- role: String（USER/ASSISTANT） // 发言角色
- content: String // 消息内容
- createdAt: DateTime // 发送时间
- 方法：
	- sanitizeContent(): String

	- toHistoryLine(): String

五、建议优先构造的“对象实例”（运行时）

（3）论坛页初始化

- commentList: List<ForumComment>

（4）个人中心初始化
- profile: UserProfile
- activities: List<UserActivityRecord>
- notifications: List<Notification>

（5）法律案件与模拟演练页初始化
- leftCaseList: List<LegalCase>（左侧案件列表）

六、模块调用关系（按用例）

1）劳动合同审查调用链
前端页面 -> AiAssistantController.analyzeContract() -> AiAssistantService.analyzeContract()
-> RiskEngineService.parseText()
-> RiskEngineService.matchRules()（调用 RiskRuleRepository 查询规则）
-> RiskEngineService.scoreRisk()
-> RiskEngineService.buildReport()（写入 AnalysisReportRepository）
-> 返回 AnalysisReport 给前端高亮展示。

2）求职黑话识别调用链
前端页面 -> AiAssistantController.detectBlackSlang() -> AiAssistantService.detectBlackSlang()
-> RiskEngineService.matchRules(category=BLACK_SLANG)
-> AnalysisReportRepository.save()
-> 返回风险片段与解释。

3）岗位风险评分调用链
前端页面 -> AiAssistantController.evaluateJobRisk()
-> AiAssistantService.evaluateJobRisk()
-> RiskEngineService.matchRules(category=JOB_RISK)
-> RiskEngineService.scoreRisk() 生成 0-100 分
-> 返回评分 + 风险等级 + 建议。

4）论坛发帖与评论调用链
前端页面 -> ForumController.createPost() -> ForumService.createPost() -> ForumPostRepository.save()
前端页面 -> ForumController.addComment() -> ForumService.addComment() -> ForumCommentRepository.save()
-> NotificationService.push(被评论用户)

5）个人中心信息维护调用链
前端页面 -> UserController.updateProfile() -> UserService.updateProfile() -> UserProfileRepository.update()
前端页面 -> UserController.getActivities() -> UserService.getUserActivities() -> UserActivityRepository.queryByUser()

6）首页案例预览调用链
前端首页 -> ContentController.listHomepageFeaturedCases()
-> CaseService.listHomepageFeaturedCases(limit=3)
-> LegalCaseRepository.queryFeatured()
-> 返回案例卡片列表。

7）案件详情 + 模拟游戏推荐调用链
前端案例详情页 -> ContentController.getCaseDetail(caseId)
-> CaseService.getCaseDetail(caseId)
-> LegalCaseRepository.findById(caseId)
前端演练入口区 -> SimulationController.getGameEntry(moduleCode)
-> SimulationService.getGameEntry(moduleCode)
-> SimulationService.listScenes(gameId)
-> 返回“案件正文 + 案例总结 + 模拟游戏入口 + 通用可切换场景列表（与案例解耦）”。

8）知识点页推荐区调用链
前端知识点页 -> ContentController.getKnowledgeRecommendations(topic)
-> CaseService.listRelatedCasesByTopic(topic)
-> SimulationService.listScenes(gameId)
-> 返回“相关案件 + 通用演练场景推荐（非必做）”。

八、实现建议（便于答辩）
- 先落地最小闭环：登录 -> 合同审查 -> 报告展示 -> 历史记录。
- 风险规则采用“数据库可配置 + 代码可扩展”双机制。
- 将 AI 能力抽象为 AiAssistantService，后续替换模型时不影响 Controller 和前端。
- 所有分析结果统一为 AnalysisReport，保证三类能力（合同/黑话/岗位）数据结构一致，便于统计与展示。

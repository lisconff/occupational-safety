第三章补充：面向对象设计（可直接并入“详细设计”）

一、设计目标与分层
结合现有文档“AI智能助手 + 求职交流论坛 + 个人主页”三大核心模块，建议采用“Controller（接口层）- Service（业务层）- Repository（数据访问层）- Domain（领域模型）”四层对象模型。

二、核心类与属性

1）用户与身份模块
（1）User
- id: String（用户ID）
- username: String（用户名）
- passwordHash: String（密码哈希）
- role: String（USER/ADMIN）
- avatarUrl: String
- email: String
- phone: String
- createdAt: DateTime
- updatedAt: DateTime

（2）UserProfile
- userId: String
- school: String
- major: String
- graduationYear: Integer
- expectedCity: String
- expectedSalary: String
- bio: String

（3）AuthSession
- token: String（JWT/会话令牌）
- userId: String
- issuedAt: DateTime
- expiresAt: DateTime
- deviceInfo: String

2）AI智能助手模块
（4）ContractDocument
- id: String
- userId: String
- sourceType: String（TEXT/FILE）
- originalText: String
- fileName: String
- fileUrl: String
- uploadedAt: DateTime

（5）RiskRule
- id: String
- ruleCode: String
- ruleName: String
- category: String（劳动合同/黑话识别/岗位风险）
- riskLevel: String（LOW/MEDIUM/HIGH）
- keywords: List<String>
- legalBasis: String（法规依据）
- suggestionTemplate: String
- enabled: Boolean

（6）RiskItem
- id: String
- reportId: String
- matchedText: String
- startIndex: Integer
- endIndex: Integer
- riskLevel: String
- reason: String
- legalBasis: String
- suggestion: String

（7）AnalysisReport
- id: String
- userId: String
- businessType: String（CONTRACT/BLACK_SLANG/JOB_RISK）
- inputSummary: String
- score: Integer（0-100）
- overallRiskLevel: String
- riskItems: List<RiskItem>
- createdAt: DateTime

（8）AiQaMessage
- id: String
- sessionId: String
- role: String（USER/ASSISTANT）
- content: String
- createdAt: DateTime

3）论坛模块
（9）ForumPost
- id: String
- authorId: String
- title: String
- content: String
- tags: List<String>
- likeCount: Integer
- commentCount: Integer
- favoriteCount: Integer
- createdAt: DateTime
- updatedAt: DateTime

（10）ForumComment
- id: String
- postId: String
- authorId: String
- content: String
- parentCommentId: String（支持回复）
- createdAt: DateTime

（11）ForumFavorite
- id: String
- userId: String
- postId: String
- createdAt: DateTime

4）个人中心模块
（12）UserActivityRecord
- id: String
- userId: String
- activityType: String（POST/COMMENT/ANALYSIS/LOGIN）
- businessId: String
- detail: String
- createdAt: DateTime

（13）Notification
- id: String
- userId: String
- type: String（SYSTEM/COMMENT/AI）
- title: String
- content: String
- isRead: Boolean
- createdAt: DateTime

三、服务类（建议）

（1）AuthService
- register(username, password)
- login(username, password)
- verifyToken(token)

（2）UserService
- getProfile(userId)
- updateProfile(userId, profileDTO)
- getUserActivities(userId)

（3）AiAssistantService
- analyzeContract(userId, textOrFile)
- detectBlackSlang(userId, text)
- evaluateJobRisk(userId, jdText)
- askAi(sessionId, question)

（4）RiskEngineService
- parseText(rawText)
- matchRules(parsedText, ruleSet)
- scoreRisk(matchedItems)
- buildReport(userId, businessType, result)

（5）ForumService
- createPost(userId, postDTO)
- listPosts(query)
- getPostDetail(postId)
- addComment(userId, postId, commentDTO)
- toggleFavorite(userId, postId)

（6）NotificationService
- push(userId, type, title, content)
- listByUser(userId)
- markRead(notificationId)

四、Repository类（建议）
- UserRepository
- UserProfileRepository
- ContractDocumentRepository
- RiskRuleRepository
- AnalysisReportRepository
- ForumPostRepository
- ForumCommentRepository
- ForumFavoriteRepository
- UserActivityRepository
- NotificationRepository

说明：Repository 仅负责 CRUD 与查询组合，不承担业务规则判断。

五、建议优先构造的“对象实例”（运行时）

（1）用户登录后初始化
- currentUser: User
- currentSession: AuthSession

（2）AI分析流程初始化
- inputDoc: ContractDocument（或临时TextInput对象）
- activeRules: List<RiskRule>
- report: AnalysisReport

（3）论坛页初始化
- postList: List<ForumPost>
- selectedPost: ForumPost
- commentList: List<ForumComment>

（4）个人中心初始化
- profile: UserProfile
- activities: List<UserActivityRecord>
- notifications: List<Notification>

六、模块调用关系（按用例）

1）劳动合同审查调用链
前端页面 -> AiAssistantController.analyzeContract() -> AiAssistantService.analyzeContract()
-> RiskEngineService.parseText()
-> RiskEngineService.matchRules()（调用 RiskRuleRepository 查询规则）
-> RiskEngineService.scoreRisk()
-> RiskEngineService.buildReport()（写入 AnalysisReportRepository）
-> 返回 AnalysisReport 给前端高亮展示。

2）求职黑话识别调用链
前端页面 -> AiAssistantController.detectBlackSlang() -> AiAssistantService.detectBlackSlang()
-> RiskEngineService.matchRules(category=BLACK_SLANG)
-> AnalysisReportRepository.save()
-> 返回风险片段与解释。

3）岗位风险评分调用链
前端页面 -> AiAssistantController.evaluateJobRisk()
-> AiAssistantService.evaluateJobRisk()
-> RiskEngineService.matchRules(category=JOB_RISK)
-> RiskEngineService.scoreRisk() 生成 0-100 分
-> 返回评分 + 风险等级 + 建议。

4）论坛发帖与评论调用链
前端页面 -> ForumController.createPost() -> ForumService.createPost() -> ForumPostRepository.save()
前端页面 -> ForumController.addComment() -> ForumService.addComment() -> ForumCommentRepository.save()
-> NotificationService.push(被评论用户)

5）个人中心信息维护调用链
前端页面 -> UserController.updateProfile() -> UserService.updateProfile() -> UserProfileRepository.update()
前端页面 -> UserController.getActivities() -> UserService.getUserActivities() -> UserActivityRepository.queryByUser()

七、关键对象关系（文字版类图说明）
- User 1..1 UserProfile
- User 1..n AnalysisReport
- AnalysisReport 1..n RiskItem
- User 1..n ForumPost
- ForumPost 1..n ForumComment
- User n..n ForumPost（通过 ForumFavorite 关联）
- User 1..n Notification

八、实现建议（便于答辩）
- 先落地最小闭环：登录 -> 合同审查 -> 报告展示 -> 历史记录。
- 风险规则采用“数据库可配置 + 代码可扩展”双机制。
- 将 AI 能力抽象为 AiAssistantService，后续替换模型时不影响 Controller 和前端。
- 所有分析结果统一为 AnalysisReport，保证三类能力（合同/黑话/岗位）数据结构一致，便于统计与展示。

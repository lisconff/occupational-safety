职途安前后端调用逻辑图

从页面交互到后端服务再到数据与AI依赖的完整链路

前端页面层（Render Static Site）

home.html

导航与入口路由，统一登录态展示。

login/register

登录注册接口调用，获取 token/userId 并缓存。

case.html

案例与场景模拟，行为埋点上报。

ai-chat-test

AI问答/文件问答，记录提问行为。

forum/profile

论坛互动、个人资料、头像和雷达图展示。

接口层（Spring Boot Controller）

AuthController

/api/auth/register
/api/auth/login

UserController

/profile /avatar /activity /risk-assessment

ForumController

帖子、点赞、收藏、评论及附件接口。

AiAssistantController

AI问答、流式响应、历史记录接口。

业务层（Service）

AuthService / JwtUtils

密码校验、JWT签发、会话写入。

UserService

资料/头像维护，行为累计，风险分计算与雷达维度生成。

ForumService / AiService

社区业务编排、AI请求中转与结果封装。

依赖与存储层

TiDB Cloud

users、user_profiles、auth_sessions、forum_* 等持久化数据。

Coze API

AI模型能力接入（通过后端统一调用）。

Render

前端静态托管 + 后端容器化部署 + 健康检查。

蓝色实线节点：核心执行单元 虚线箭头：请求流向（前端 -> 后端 -> 依赖） 结果回流：统一 JSON 响应并更新页面状态
建议演示顺序：登录 -> 案例学习 -> AI提问 -> 论坛浏览 -> 个人中心雷达图，能完整体现行为数据上报与风险评估闭环。
